# AUDIT.md — hela-ai-blog

## File Map
- `app/` — Next.js App Router pages (layout, home, post slug, drafts, RSS, sitemap)
- `components/` — GiscusComments, PostCard UI components
- `content/posts/` — MDX blog posts (8 published, drafts folder)
- `lib/` — posts.js (MDX parsing, gray-matter frontmatter)
- `public/` — static assets
- `out/` — static export (15 pages, generated by `next build`)
- `next.config.js` — output: 'export' + MDX support
- `SECURITY_REVIEW.md` — Seth's full security review (2026-03-24)

## Key Entry Points
- `app/layout.js` — root layout, fonts, metadata
- `app/page.js` — homepage, lists published posts
- `app/posts/[slug]/page.js` — individual post renderer (MDX via next-mdx-remote)
- `app/drafts/page.js` — internal drafts listing (no auth — public if deployed)
- `lib/posts.js` — `getPublishedPosts()`, `getDraftPosts()`, `getPostBySlug()`

## Threat Model
- Static site (Next.js `output: 'export'`) — no server-side execution at runtime
- Hosted on Cloudflare Pages — edge CDN, DDoS protection by default
- Giscus comments load external script from `giscus.app` (trusted, but no SRI pinning)
- No auth on `/drafts` route — drafts are publicly accessible if route exists in build
