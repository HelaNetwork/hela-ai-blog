# HeLa AI Blog — Design Document

## Architecture

Static Next.js 14 site (App Router) with MDX content, exported to static HTML via `output: 'export'`. Deployed to Cloudflare Pages via Wrangler. No server-side runtime — all rendering is at build time.

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Framework | Next.js 14 (App Router) |
| Content | MDX via `next-mdx-remote`, frontmatter via `gray-matter` |
| Styling | Tailwind CSS 3 + `@tailwindcss/typography` + custom CSS variables |
| Fonts | Press Start 2P (headings), VT323 (body) — Google Fonts |
| Design | Retro pixel-art theme (see `public/preview.html` for reference) |
| Images | PIL-generated 800x400 post headers in `public/images/posts/` |
| Dates | `date-fns` |
| Deployment | Cloudflare Pages (via `wrangler`) |
| Comments | Giscus (GitHub Discussions, external script) |
| Export | `next build` → static `out/` directory |

## Design System (2026-03-30 — pixel-art restyle)

**Colors:** `--bg #080810`, `--panel #0f0f1c`, `--card #13131f`, `--border #252540`, `--accent #C6F135` (lime), `--accent2 #ff3c6f` (pink), `--accent3 #ffe94d` (yellow), `--accent4 #3cf0ff` (cyan), `--text #c8c8e8`, `--dim #5a5a7a`

**Components (CSS classes in globals.css):** `.pixel-nav`, `.ticker`, `.pixel-card`, `.post-tag`, `.char-slot`, `.ground-line`, `.pixel-footer`, `.pixel-bar`

**Header images:** Generated by Python PIL script (documented in AUDIT.md). Each 800x400 image: dark grid background + character PNGs from `~/projects/media-factory/output/pixel-chars-poses/{name}/{pose}.png` scaled to 192px, accent bar at top. Saved to `public/images/posts/{slug}.png`.

## Content Structure

Posts are MDX files in `content/posts/`. Frontmatter fields: `title`, `date`, `author`, `tags`, `summary`, `image`. Posts are sorted by date descending. No database — all content is filesystem.

`lib/posts.js` reads files at build time with `fs.readdirSync` and `gray-matter`. No try/catch around file reads (build-time risk only; no runtime impact).

## Data Flow

1. `next build` → `lib/posts.js` reads `content/posts/*.mdx`
2. `getAllPosts()` returns sorted post metadata array
3. `getPostBySlug(slug)` returns full post content for dynamic `[slug]` route
4. Static HTML generated for all slugs at build time
5. `out/` directory uploaded to Cloudflare Pages via `wrangler pages deploy out/`

- **Deploy:** `wrangler pages deploy out/ --project-name hela-ai-blog`

## SEO Setup

- `export const metadata` in each page file (Next.js App Router pattern)
- `app/sitemap.js` — generates `/sitemap.xml` at build time
- `app/rss/` — generates RSS feed
- `trailingSlash: true` in next.config.js for Cloudflare Pages URL compatibility

## Known Risks

- `/drafts` route is publicly accessible — no auth gate
- Giscus comment script loaded from `giscus.app` with no SRI hash
- No CSP headers configured
- `furykc` GitHub username exposed in Giscus config (links to KC's private account)
- `lib/posts.js` has no error handling on file reads
